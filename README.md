## Возможности API

### 1. Получение гексагонов по родителю

**`GET /hex`** – возвращает гексагоны исходного датасета, входящие в указанный `parent_hex`.

**Параметры:**

| Параметр     | Тип | Описание                       |
| ------------ | --- | ------------------------------ |
| `parent_hex` | str | Индекс родительского гексагона |

**Пример запроса:**

```http
GET http://localhost:8000/hex?parent_hex=8a11aa648367fff
```

**Пример ответа:**

```json
[
  ["8c11aa6483601ff", -120, 93],
  ["8c11aa6483603ff", -119, 94]
]
```

---

### 2. Медианное значение по гексагонам

**`GET /avg`** – возвращает массив гексагонов заданного разрешения с медианным значением `level`, сгруппированным по `cell_id`.

**Параметры:**

| Параметр     | Тип | Описание             |
| ------------ | --- | -------------------- |
| `resolution` | int | Разрешение H3 (0–12) |

**Пример запроса:**

```http
GET http://localhost:8000/avg?resolution=0
```

**Пример ответа:**

```json
[
  ["8011fffffffffff", -62, 1],
  ["8011fffffffffff", -63, 2]
]
```

---

### 3. Элементы внутри полигона

**`GET /bbox`** – возвращает элементы исходного датасета, полностью находящиеся внутри заданного полигона.

**Параметры:**

| Параметр | Тип | Описание                               |
| -------- | --- | -------------------------------------- |
| `border` | str | Массив координат lat/lon через запятую |

**Пример запроса:**

```http
GET http://localhost:8000/bbox?border=56.035953/37.911440,56.280315/37.589786,56.0/37.182514
```

---

### 4. Экспорт гексагонов в KML

**`GET /bbox_kml`** – аналогично `/bbox`, но возвращает файл `.kml` для загрузки.

**Пример запроса:**

```http
GET http://localhost:8000/bbox_kml?border=56.035953/37.911440,56.280315/37.589786,56.0/37.182514
```

## Файл конфигурации `config.py`

Сервис использует `config.py` для централизованного хранения настроек.
Файл основан на Pydantic и содержит настройки координат, разрешения данных, параметров логирования и смещений для вычислений.

**Основные параметры:**

| Параметр             | Описание                                 | Значение по умолчанию                                                      |
| -------------------- | ---------------------------------------- | -------------------------------------------------------------------------- |
| `CENTER_LAT`         | Центральная широта                       | 56.0                                                                       |
| `CENTER_LON`         | Центральная долгота                      | 38.0                                                                       |
| `RADIUS_KM`          | Радиус области анализа в км              | 7.0                                                                        |
| `DATASET_RESOLUTION` | Разрешение H3 для исходного датасета     | 12                                                                         |
| `LEVEL_MOD`          | Модуль для вычислений уровня             | 74                                                                         |
| `LEVEL_OFFSET`       | Смещение для вычислений уровня           | -120                                                                       |
| `CELL_ID_MOD`        | Модуль для вычислений cell_id            | 100                                                                        |
| `CELL_ID_OFFSET`     | Смещение для вычислений cell_id          | 1                                                                          |
| `LOG_LEVEL`          | Уровень логирования                      | "DEBUG"                                                                    |
| `LOG_FORMAT_STDOUT`  | Формат логов для консоли                 | цветное форматирование с временем, уровнем, функцией, строкой и сообщением |
| `LOG_FILE_PATH`      | Путь к файлу логов                       | `logs/app.log`                                                             |
| `LOG_ROTATION`       | Максимальный размер файла перед ротацией | "10 MB"                                                                    |
| `LOG_RETENTION`      | Срок хранения логов                      | "10 days"                                                                  |
| `LOG_COMPRESSION`    | Формат сжатия архивов логов              | "zip"                                                                      |

## Установка

### Клонирование репозитория

```bash
git clone git@github.com:status4O4/h3_service.git
cd h3_service
```

### Создание виртуального окружения (рекомендуется)

```bash
python -m venv venv
source venv/bin/activate  # Linux / macOS
venv\Scripts\activate     # Windows
```

### Установка зависимостей через pip

```bash
pip install .
```

> Все зависимости указаны в `pyproject.toml`.

---

## Запуск сервера

### Через uvicorn (рекомендуемый способ)

```bash
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000 
```

* `--reload` – автоматическая перезагрузка при изменениях кода.
* `--host 0.0.0.0` – доступ из сети (можно оставить `127.0.0.1` для локального использования).

## Структура проекта

```text
├── README.md              # документация и примеры использования
├── pyproject.toml         # зависимости и метаданные проекта
├── app
│   ├── main.py            # точка входа FastAPI сервера
│   ├── config.py          # настройки приложения (H3, координаты, логирование)
│   ├── dependencies.py    # зависимости FastAPI через Depends
│   ├── api
│   │   ├── routes.py      # маршруты API (/hex, /avg, /bbox, /bbox_kml)
│   │   └── schemas.py     # Pydantic модели запросов/ответов
│   ├── domain
│   │   └── models.py      # бизнес-модели гексагонов
│   ├── infrastructure
│   │   ├── dataset.py     # загрузка и подготовка датасета
│   │   ├── repository.py  # основные операции с гексагонами
│   │   └── uow.py         # Unit of Work для согласованного доступа к данным
│   ├── services
│   │   └── hex_service.py # бизнес-логика работы с гексагонами
│   └── utils
│       ├── geometry.py    # геометрические операции
│       ├── kml.py         # генерация KML
│       └── logger.py      # настройка логирования
```
